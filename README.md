# CLAUDE.md -- полное руководство по настройке AI-агентов

Руководство по заполнению CLAUDE.md для Claude Code агентов.
Основано на боевом опыте настройки мультиагентной системы из 6 агентов в продакшене.

## Что такое CLAUDE.md

CLAUDE.md -- главный конфигурационный файл Claude Code агента.
Это не просто инструкция. Это **конституция**, по которой агент принимает решения,
расставляет приоритеты и определяет границы дозволенного.

Без CLAUDE.md агент -- болванка. С CLAUDE.md -- полноценный сотрудник.

## Два уровня

```
~/

│   ├── CLAUDE.md              <- ГЛОБАЛЬНЫЙ (для всех агентов)
│   └── rules/
│       ├── python.md
│       ├── typescript.md
│       └── bash.md
│
└── .claude-lab/
    └── agent-1/
        └── .claude/
            ├── CLAUDE.md      <- РАБОЧИЙ (для конкретного агента)
            ├── core/
            │   ├── USER.md
            │   ├── rules.md
            │   ├── warm/decisions.md
            │   └── hot/handoff.md
            └── tools/TOOLS.md
```

### Глобальный (`~/.claude/CLAUDE.md`)

Конституция. Применяется ко ВСЕМ агентам на машине.
Содержит: языковые правила, стиль кода, git-правила, принципы безопасности, 9 принципов работы.

### Рабочий (`.claude/CLAUDE.md`)

Должностная инструкция конкретного агента.
Содержит: роль, характер, стиль общения, память, координация с другими агентами.

## Ключевые паттерны

1. **Приоритет правил:** Безопасность > Указание оператора > Проверка фактов > Границы > Стиль
2. **Тишина = баг.** Оператор всегда должен видеть, что делает агент
3. **Обучение -> система.** Ошибка должна становиться системным изменением, а не записью в памяти
4. **Agent Native.** Бэкенд -- для агентов (API). Фронтенд -- для людей
5. **Доверяй, но проверяй.** База данных > реальные проверки > локальная память
6. **Экономия токенов.** Внутренние файлы на английском (2--3x меньше токенов), максимум 4 @includes, авто-компакт 400K
