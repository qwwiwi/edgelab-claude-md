01 Why agents need memory

Every AI session starts with a blank slate. The model remembers nothing from the last conversation -- not decisions, not preferences, not even your name.

Memory is a set of structured files the agent reads at boot, restoring full context before your first message.

Analogy
Memory is a notebook on your desk. Every morning you open it, read yesterday's notes, and pick up where you left off.

02 The session lifecycle

PhaseWhat happensMemory role
BootLoad system prompt + memory filesIdentity, context, rules
ActiveConversation within context windowWorking memory (in-context)
SaveSession ends, key facts extractedWrite to persistent storage
Next bootNew session loads updated filesContinuity restored
Key concept
Context windows are finite: 200K tokens for Claude, 128K for GPT. Anything beyond that is lost unless persisted to files or a database.

03 Memory layers

Layer 1 -- Identity (always loaded)

Who the agent is, who the user is, core rules. Small stable files: SOUL.md, USER.md, RULES.md.

Layer 2 -- Warm (recent decisions)

Last 14 days of decisions and active projects. Always in context, rotated regularly.

Layer 3 -- Cold (on-demand)

Historical facts, past decisions. Loaded only when the agent needs to look something up.

Layer 4 -- Semantic (vector DB)

When files exceed 50+, vector search indexes all content and retrieves by meaning.

Layer 1: Identity  [always loaded, ~2K tokens]
Layer 2: Warm      [always loaded, ~4K tokens, rotated 14d]
Layer 3: Cold      [on-demand read, unlimited]
Layer 4: Semantic  [vector search, cross-session]

04 File-based memory

workspace/
├── SOUL.md        # agent identity, style, principles
├── USER.md        # user profile and preferences
├── RULES.md       # operational boundaries
├── MEMORY.md      # long-term facts and decisions
├── decisions.md   # recent decisions (warm layer)
├── handoff.md     # last session context (hot)
└── memory/        # date-based journals (cold)
    └── 2025-01-15.md
Best practice
Keep identity files under 200 lines each. Every character costs tokens -- the agent reads these at every session start.
Common mistake: secrets in memory files
Never store API keys or tokens in memory files. Store paths to secret files instead. If the workspace ever leaks, keys stay safe.

05 Vector search

When your agent has hundreds of files, reading them all is impossible. Vector search converts text into embeddings and finds relevant content by meaning.

ModeHow it worksBest for
Keyword (BM25)Exact word matchingKnown terms
VectorSemantic similarityFuzzy queries
HybridKeyword + vector + rerankBest quality
 
# Index memory files
memory-index update ./workspace/memory/

# Search by meaning
memory-index query "what auth method did we choose?"
# [0.92] decisions.md: "JWT auth over session cookies"
# [0.85] MEMORY.md: "Auth: JWT RS256, 15min expiry"
When to add vector search
Start with file-based memory. Add vector search at 50+ files. Premature optimization adds complexity without value.
